- **주제 : `map` 메서드와 `filter` 메서드를 이용해 props를 넘겨 화면 렌더링하기**
- 학습 목표
    1. 2일차에서 배운 map, filter를 이용한 **데이터 가공**을 컴포넌트 단위로 이해한다.
    2. Props의 사용법에 대해 이해한다.
    
- 복습할 개념 체크 리스트
    - [ ]  **React가 데이터를 관리하는 방식**
     단방향 데이터 흐름(One-way data flow)과 상태 중심 렌더링 개념 기반
     데이터가 부모에서 자식으로만 흐른다.
     자식이 직접 데이터를 변경할 수 없다.
     직접 DOM을 조작할 필요가 없다.
     상태가 바뀌면 자동으로 다시 렌더링(rendering) 해준다.
     전역 데이터 저장소 -> 어디서든 접근 가능

    - [ ]  **Props의 개념과 사용하는 방법**
     properties의 줄임말
     React에서 컴포넌트는 함수처럼 동작하는데 그 매개변수 역할을 하는게 props다.
     props는 읽기 전용!

    - [ ]  **React에서 map으로 컴포넌트를 렌더링 하는 방법**
     map()은 배열을 순회하여 새로운 배열을 반환한다.
     각 요소를 구분하기 위한 key는 필수!
     부모가 배열을 가진 상태에서 map으로 자식을 여러 개 생성하여 렌더링하는 것이 가능하다.
     props로 함수도 전달하면 이벤트 핸들링도 가능하다.

    - [ ]  **React에서 filter로 컴포넌트를 필터링 하는 방법**
     filter()는 조건에 맞는 항목만 화면에 보여줄 때 사용하는 배열 메서드이다.
     map()이 모든 요소를 렌더링한다면, filter()는 필요한 것만 걸러서 렌더링 한다.
     배열을 조건식으로 걸러서 새로운 배열을 만든 후, 그 배열을 map으로 렌더링할 때 filter() 사용
     1. filter()로 조건에 맞는 요소만 남긴다.
     2. map()으로 jsx 형태로 변환 후 렌더링 한다.

- **문제 요구 사항**

- `CardList.jsx` 컴포넌트 내 요구 사항
    - 카드의 리스트 UI를 담당하는 컴포넌트 입니다.
    - `listData`를 받아와서 카드 리스트를 렌더링 해야 합니다.
    - `listData`의 각 요소를 `Card.jsx` 컴포넌트에 props로 넘겨주어야 합니다.
    - `Card.jsx` 컴포넌트에는 `item`을 넘겨주어야 합니다.
    
- `Card.jsx` 컴포넌트 내 요구사항
    - 카드의 1개의 UI를 담당하는 컴포넌트 입니다.
    - `item`을 받아와서 카드를 렌더링 해야 합니다.
    - `item`의 `title`, `description`, `category`를 알맞는 html 태그에 넣으세요.
    - `item`은 `CardList.jsx`에서 전달됩니다.
